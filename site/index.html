mycel

search, grown like a fungus.

Every node is a complete search engine — polite crawler, WARC archive, tantivy index, BM25 × centrality ranking, HTTP API. Fully useful with zero peers. Federation is additive: dial peers you trust by public key, fan out queries, exchange crawl corpora. No DHT. No tokens. No cloud.

the engine

Complete at n = 1.

A mycel node with zero peers is not a degraded mode — it is the product. One process runs the entire pipeline over the hosts you choose, and every stage is inspectable with standard tools: sqlite3, zcat, plain HTTP.

Two pens, two owners.

Two dedicated OS threads own the two single-writer resources; everything else is tokio. The db-writer thread holds the only SQLite write connection and the open WARC shard — every state change is one ordered command stream, drain-batched into single transactions. The indexer thread owns the tantivy writer and the near-duplicate gate.

Crash safety is a protocol, not a hope: a page is fsync'd into WARC and its database row lands in the same transaction that advances the shard watermark. On boot, the shard truncates back to the watermark. No torn writes, no phantom pages.

what's in the binary

Boring dependencies, opinionated plumbing.

🤝

Polite by construction

One in-flight request per host, enforced by the claim query — not by good intentions. RFC 9309 robots, sticky 429 backoff that only ever rises, and a crawler that refuses to start until crawl.contact_url identifies you in the user agent.

🗄

WARC is the truth

Every fetch lands in standard .warc.gz — the same format as the Internet Archive and Common Crawl. SQLite and the tantivy index are derived state, rebuildable from the archive at any time. Peers exchange shards, never index segments.

📖

Ranking you can read

score = bm25 × (1 + 0.3 × centrality). Text relevance from tantivy; host-level harmonic centrality from your own webgraph with HyperBall. Explainable end to end — no embeddings, no oracle.

🔑

Federation without ceremony

iroh QUIC: dial peers by ed25519 public key, ~90% direct through NAT, relays as fallback. The allowlist is the only auth gate. preset = "empty" runs a LAN or airgapped mesh with zero external infrastructure.

🌍

Bootstrap from Common Crawl

Don't crawl the planet — it's been crawled. Feed mycel two CSVs and it pulls ranged GETs from data.commoncrawl.org: throttled, resumable, seeded with real host centrality, straight into your own WARC store.

🔧

Hand-rolled where it counts

The WARC writer is ~380 lines. CLI parsing is by hand. The search UI is format!-ed HTML. 27 dependencies, chosen to be boring — the spec lists every one, and every deliberate absence (no clap, no chrono, no anyhow…).

federation

Trust is social, not cryptographic-economic.

There is no open network to join and no token to stake. Each node lists the peers it accepts; that allowlist, checked after the QUIC handshake, is the single gate. Exchange node ids out of band — like SSH keys, like phone numbers.

Results interleave round-robin, deduped by URL — never a cross-node score sort, because BM25 scores from different corpora aren't comparable, so mycel doesn't pretend they are.

Two verbs, one protocol.

Query fan-out — search --federated asks every peer in parallel behind a hard timeout. Slow peers get dropped, not waited on.

Shard sync — pull a peer's crawl corpus as immutable, blake3-verified WARC shards: quota-capped, resumable, self-origin only. No transitive flooding — you replicate the peers you chose, not the peers they chose.

# mycel.toml
[federation]
enabled = true

[[federation.peers]]
id   = "<bee's 64-hex endpoint id>"
name = "bee"   # result badge
sync = true    # pull bee's corpus

mycel peers check proves dial + auth + protocol for every peer in one round trip. A peerless node binds no sockets and publishes nothing.

durability

The index is never precious.

WARC is the source of truth; everything else is derived state. Recovery is a normal code path, not a disaster plan — the same ingest and reindex you'd use any other day.

Index corrupt?

Or tantivy upgraded across versions. Delete it. It was always disposable.

rm -rf <data>/index
mycel reindex

Database lost?

WARC intact means nothing is lost. Rebuild the catalog, then the index.

mycel ingest <data>/warc
mycel reindex

Backups?

Two things, and pointedly not the index — it's derived state; never back it up.

sqlite3 mycel.sqlite ".backup …"
rsync -a warc/ backup/warc/

decisions, not gaps

What mycel refuses to do.

Every feature below was considered, researched, and rejected with written evidence. The refusals are binding: they live in SPEC.md §9, defended in RESEARCH.md.

  • no.

    JavaScript rendering

    Pages are indexed as served. A headless browser farm is a different product with a different cost curve.

  • no.

    Embeddings & vector search

    BM25 × host centrality is cheap, explainable, and good. Semantic search adds a GPU bill and an oracle you can't audit.

  • no.

    DHT & peer discovery

    You list your peers; the allowlist is the gate. Open networks inherit open-network spam, sybils, and poisoning.

  • no.

    Trustless peering

    No reputation systems, no staking, no "token anything". Trust is social — the same way you trust an SSH key.

  • no.

    Custom storage formats

    SQLite, WARC, tantivy. Every byte on disk is inspectable with sqlite3 and zcat, and outlives this project.

quickstart

Grow your own.

1 · Install

Prebuilt tarballs for Linux (x86_64, aarch64) and macOS (Apple Silicon) are on the releases page — one static-ish binary, drop it on your PATH. Or build from source (Rust ≥ 1.85):

git clone https://github.com/splch/mycel
cd mycel && cargo build --release

2 · Crawl & search

mycel init                  # config, data dir, identity
$EDITOR mycel.toml          # set crawl.contact_url
mycel seed blog.example.org
mycel crawl --limit 500
mycel search "some phrase"
mycel run                   # daemon + API on :8080

An empty mycel.toml is valid — every setting has a default. Then open http://127.0.0.1:8080 for the built-in search UI, or hit /api/search?q=… for JSON.
